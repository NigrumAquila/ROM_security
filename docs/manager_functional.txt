Программа является управляющим звеном. В ней реализован необходимый функционал для развертывания и управления базой данных внешних запоминающих устройств.


1 - Добавление нового запоминающего устройства. Эта опция добавляет устройство в выбранную базу данных, а также создает на внешнем запоминающем устройстве файл с идентификатором для опознования программой контроля. При совпадении нового сгенерированного идентификатора с уже существующим в базе даных, новый идентификатор перегенерируется до тех пор, пока совпадение не будет отброшено. Сохранять файл с идентификатором для внешнего устройства следует в корне внешнего запоминающего устройства. Если файл с идентификатором устройства существует, в него добавится еще одна строка, для дополнительной базы данных, иначе файл будет создан и в него запишется сгенерированный идентификатор. При выборе этой функции необходимо указать путь до базы данных в диалоговом окне. Далее указать желаемый путь установки идентификатора для внешнего запоминающего устройства.


2 - Удаление внешнего запоминающего устройства из базы данных. При выборе этой функции необходимо указать путь до базы данных в диалоговом окне. Далее указать номер уникального идентификатора в базе даных, которому соответствует идентификатор удаляемого запоминающего устройства. В случае, когда идентификатор не существует в базе данных, выводится соответствующее сообщение. Функция работает только для файлов с расширением ".db".


3 - Проверка целостности базы данных. При выборе этой функции необходимо указать путь до базы данных в диалоговом окне. В случае, если файл в порядке, будет выведено соответствующее сообщение. Если база данных повреждена, будет выведено сообщение об этом с номером поврежденной строки в базе данных. Проверка осуществляется до поврежденной строки для экономии ресурсов. Проверка включает в себя контроль уникальности идентификаторов, контроль паттерна: "идентификатор_БД: идентификатор_устройства". Функция работает только для файлов с расширением ".db".


4 - Вывод информации о базе данных. При выборе этой функции необходимо указать путь до базы данных в диалоговом окне. Показывает дату создания файла, дату изменения файла, идентификаторы запоминающих устройств, записанных в базу данных. Функция работает только для файлов с расширением ".db".


5 - Создание файла базы данных. При выборе этой функции происходит запрос желаемого имени для базы данных у пользователя через диалоговое окно. При вводе данных, создается файл с соответствующим названием и расширением '.db'.


6 - Создание резервной копии базы данных. При выборе этой функции необходимо выбрать файл с желаемой базой данных для копирования в диалоговом окне, далее в следующем диалоговом окне указать путь сохранения и имя файла резервной базы данных. Функция работает только для файлов с расширением ".db".
